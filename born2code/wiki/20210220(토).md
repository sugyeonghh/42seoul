#### 1. MiniLibX

* `pixel` 을 찍어서 도형 만들기

  * 사각형 만들기

    * 코드

    ```c
    void	draw_square(t_data *img, int x, int y, int size, int color)
    {
    	int	i;
    
    	i = -1;
    	while (++i < size)
    	{
    		my_mlx_pixel_put(img, x + i, y, color);
    		my_mlx_pixel_put(img, x + i, y + size - 1, color);
    	}
    	i = size;
    	while (--i > 0)
    	{
    		my_mlx_pixel_put(img, x, y + i, color);
    		my_mlx_pixel_put(img, x + size - 1, y + i, color);
    	}
    }
    ```

      void	draw_circle(t_data *img, int x, int y, int r, int color)
      {
      	int	a;
      	int	b;
      
      	a = x - r - 1;
      	while (++a <= x + r)
      	{
      		b = y - r - 1;
      		while (++b <= y + r)
      		{
      			if (pow(r, 2) <= pow(a - x, 2) + pow(b - y, 2) + 2000 && pow(r, 2) >= pow(a - x, 2) + pow(b - y, 2) - 100)
      				my_mlx_pixel_put(img, x + a, y + b, color);
      		}
      	}
      }

## 7. 학습 내용에 대한 개인적인 총평

* 클러스터에서 할 때는 `mlx.h` 를 설치하지 않아도 바로 돼서 내장 함수인 줄 알고 있었는데, 집에 와서 맥북으로 헤더를 `include` 했더니 오류가 생겼다. 그래서 어떻게 해야하는 건지 몰라서 거기에 시간을 많이 쏟았다. 클러스터에 있으면 이런 고민을 하지 않아도 되는데 못가니까 짜잘한 거에 시간을 쏟는게 너무 아깝고 아쉽다. 
* 원 그리는 걸 어떻게 해야하나 싶었는데 원의 방정식을 이용해서 생각보다 쉽게 그렸다(이게 맞는 것인지는 모르지만). 픽셀로 아무것도 그리지 못했을 때는 너무 우울했는데 막상 원이나 사각형을 그리고 나니까 너무 신난다! 나는 확실히 눈에 보이는 걸 만드는 것을 더 좋아하는 것 같다. `html` 할 때도 눈에 바로바로 보여서 신났는데 지금도 너무 신난다. 하지만 `cub3D` 를 구현하는 건 재미없겠지... 엄청 어렵겠지.... 잘하고싶다... 화이팅
