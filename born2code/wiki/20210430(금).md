## 6. 상세 학습 내용

#### 1. Docker

![docker-image](https://user-images.githubusercontent.com/55546817/116815520-ddd47880-ab98-11eb-88ef-17cf89917172.png)

* 도커: 컨데이너 기반의 오픈소스 가상화 플랫폼
* 컨테이너: 이미지를 실행한 상태
* 이미지: 컨테이너 실행에 필요한 파일과 설정값 등을 포함하고 있는 것 (프로그램 실행에 필요한 모든 것)
  * 새로운 서버가 추가되면 미리 만들어 놓은 이미지를 다운받아서 컨테이너를 생성하면 됨



<hr>

#### 2. Docker hub

* 공개 이미지를 서버에 저장하고 무료로 관리해주는 시스템
* 공개된 이미지를 다운받아서 사용하거나 저장소를 직접 만들어서 관리할 수 잇음



<hr>

#### 3. Docker file

* Makefile 처럼, 필요한 패키지를 설치하고 동작하기 위한 설정을 담은 파일
* 명령어 순서에 따라 이미지를 빌드
* 이미지가 계층적으로 쌓이면서 생성 (layer 구조)
* 명령어
  * FROM: 유효한 도커파일 시작 명령어
  * LABLE: 이미지 상세 정보 작성을 위한 명령어
  * RUN: 명령을 실행
  * COPY: 호스트 OS 의 파일 or 디렉토리를 컨테이너 안의 경로로 복사
  * EXPOSE: 해당 컨테이너가 런타임에 지정된 네트워크 포트에서 수신 대기중임을 알림
  * CMD: 생성된 컨테이너를 실행시킬 명령어



<hr>

#### 4. 데비안

* 리눅스 OS 의 종류



<hr>

#### 5. Nginx

* 오픈소스 웹 서버 프로그램
* 규모가 작고 정적 데이터 처리가 많은 서비스에 적합
